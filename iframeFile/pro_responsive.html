三、响应式开发原理

  • 就是使用媒体查询针对不同宽度的设备进行布局和样式的设置，从而适配不同设备的目的
  • 屏幕
    • 屏幕像素
      超小屏幕（手机）<768px
      小屏设备（平板）>=768px ~ <992px
      中等屏幕（桌面显示器）>=992px ~ <1200px
      宽屏设备（大桌面显示器）>=1200px

四、响应式布局容器

响应式需要一个父级做为布局容器，来配合子级元素来实现变化效果。 ,原理就是在不同屏幕下，通过媒体查询来改变这个布局容器的大小，再改变里面子元素的排列方式和大小，从而实现不同屏幕下， 看到不同的页面布局和样式变化。

  • 就是使用媒体查询针对不同宽度的设备进行布局和样式的设置，从而适配不同设备的目的
  • 屏幕
    • 平时我们的响应式尺寸划分
      屏幕像素
      超小屏幕（手机，小于768px）：设置宽度为100%
      小屏幕（平板，大于等于768px）：设置宽度为750px
      中等屏幕（桌面显示器，关于等于992px）：设置宽度为970px
      大屏幕（大桌面显示器，大于等于1200px）：设置宽度为1170px
      其他（大于等于1280px）设置宽度为1280px
  • 代码
    • Css 
        @media screen and(max-width:767px){.container{width: 100%;}}
  @media screen and(min-width:768px){.container{width: 750px;}}
  @media screen and(min-width:992px){.container{width: 970px;}}
  @media screen and(min-width:1200px){.container{width: 1170px;}}

五、案例

  • 当我们屏幕大于等于768像素，我们给布局容器container宽度为750px。
  • container里面包含8个小i盒子，每个盒子的宽度定为93.75px（750/8），高度为30px，浮动一行显示
  • 当我们屏幕缩放，宽度小于768像素的时候，container盒子宽度修改为100%宽度。
  • 此时里面的8个小i，宽度修改33.33%，这样一行就只能显示3个i，剩余下行显示
  • 代码
    • Html 
        <style>
      * {margin: 0;padding: 0;}
      ul {list-style: none;} 
      .container {width: 750px;margin: 0 auto;}
      ul li {float: left;width: 93.75px;height: 30px;background-color: skyblue;}
      @media screen and (max-width: 767px) {
        .container {width: 100%;}
        .container ul li {width: 33.33%;}
      } 
  </style>

  <div class="container">
      <ul >
          <li>导航1</li>
          <li>导航2</li>
          <li>导航3</li>
          <li>导航4</li>
          ...
      </ul>
  </div>

八、媒体查询

  • 写法
    • Css 
        @media (orientation : portrait ){....}     /* 媒体查询 （媒体的特性 ： 垂直方向）｛执行代码｝ */

  /* 比如判断屏幕最大为480px */
  @media ( max - device-width : 480px){...}  /* 如果屏幕小于480px 那么就会返回一个true */
  • 属性和属性值
    • 代码描述
      aspect-ratio ： 3/2可视窗口宽高比
      device-aspect-ratio: 3/2设备可视窗口宽高比
      orientation : portrait垂直方向
      orientation : landscape水平方向
      maximum ：最大的，相当于小于或者等于
      minimum ：最小的，相当于大于或者等于
      max-height / width： 480px当网页可视窗口小于等于 480px 为true 时，触发后面代码
      min-height / width当网页可视窗口大于等于
      device-max-height / width： 480px当设备小于等于 480px
      device-min-height / width当设备大于等于
  • 多重写法 逗号隔开
    • Css 
        @media (orientation : portrait)，@media screen and (max-widht:480px) {
      ....
  }  

  @media screen and(min-device-width:200px) and(max-device-width:300px){
      /* 执行代码 */
  }
  • not
      @media not screen {
      /* 如果设备不是 screen 才执行{ }里面的代码 */
  }
  • only
      @media only screen {
      /* 如果设备支持媒体查询才会引用这里面的代码 */
  }
  • 图片
    1. img 要响应式 设置宽度100%，如果不被拉伸 需要设置 max-width:100%，这样会跟着容器的宽度走
    2. 背景图片设置：-webkit-background-size:100%; (兼容性) background-size:100%;
    3. 造成问题，当缩小屏幕的时候，图片和上面区域会留下很多空白
  • 引入方式
    • Html 
        1、外部引入 
  <!-- 当屏幕大于300 且小于500px 的时候，会引入index1.css的样式 -->
  <link rel="stylesheet" href="./css/index1.css" media="(min-device-width:300px) and (max-device-width:500px)">
  <!-- 当屏幕大于100 且小于300px 的时候，会引入index2.css的样式 -->
  <link rel="stylesheet" href="./css/index2.css" media="(min-device-width:100px) and (max-device-width:300px)">
                  
  2、内部样式 
  <style> media="(min-device-width:300px) and (max-device-width:500px)"> /* 代码 */ </style>
